## Jonathan Staude

I build AI agents and the infrastructure they run on — founder of **[11data](https://11data.ai)**.

My focus is the unglamorous layer that makes agents actually useful in production: durable
memory, a documentation-first way of working, and the tooling around the loop. I care more
about systems that stay reliable over months than demos that look good for an afternoon.

### What I'm building

- **[LongMem](https://github.com/jonameron/longmem-oss)** — long-term memory for AI agents.
  One API call to store a memory, one to recall it, with hybrid semantic + keyword search.
  Open SDKs and connectors live in **[longmem-sdk](https://github.com/11data/longmem-sdk)**.
- **[docs-driven-agents](https://github.com/jonameron/docs-driven-agents)** — the
  documentation-first methodology I use to build with coding agents: docs as the source of
  truth, procedures codified as reusable skills, wired into a loop that gets more reliable
  the more it runs.
- **[11data](https://11data.ai)** — where this work ships. I also bring on student interns
  and put their work out in the open (see
  [11data_agent_research](https://github.com/jonameron/11data_agent_research)).

### Background

I've been building community-driven, decentralized systems since well before it was a
category — back in 2018 I led
[protchain](https://github.com/jonameron/MK-Protection), a decentralized cellphone-insurance
concept presented at D1Conf in Prague. The throughline from then to now is the same: take a
process everyone assumes needs a central operator, and figure out how to distribute it.
